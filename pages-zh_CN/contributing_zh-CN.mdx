# 为 Madara 贡献

Madara 与其他技术栈的一个主要区别在于其文化。从第一天起，Madara 就是一个开源技术栈，任何人都可以做出贡献。Madara 开发者并不局限于任何一家公司，截至撰写本文时，Madara 拥有 **90+ 位贡献者**，遍布全世界。此外，开源并不意味着你只是出于个人兴趣做出贡献（尽管这当然非常重要！）。在 Madara 中的每一项贡献都能通过 [OnlyDust](https://app.onlydust.com/p/madara) 获得报酬！

![contributing](../static/img/contributing.png)

> 注意：目前，包括 Starkware、Pragma、Karnot 和 Kasar Labs 在内的多家公司都参与维护 Madara。然而，这并不限制新的个人和公司加入并成为核心贡献者。随着时间的推移，我们对代码质量提出了更高的要求，以确保技术栈的稳健和可维护性。

## 贡献步骤

<Steps>

### 找到问题

首先，如果您是新手，请浏览[贡献者入门包](https://github.com/keep-starknet-strange/madara/blob/main/docs/contributor-starter-pack.md)。这份文档详细说明了开始向 Madara 贡献所需的先决条件。一旦完成了这些步骤，就可以开始深入研究了！在 Madara 的 Github [这里](https://github.com/keep-starknet-strange/madara/issues)有很多问题等待解决。浏览这些问题，寻找您感兴趣的内容。您可以搜索 `good-first-issues` 来找到一些更容易开始的问题。

### 评论问题

现在您已经找到了引起您注意的问题，是时候深入挖掘了。如果您有任何疑问，可以在问题本身下方评论，作者或维护者会回应您。同时，随时在 [Madara 电报群](https://t.me/MadaraStarknet) 发送您的问题，以获得更快的回复。一旦您决定对该问题做出贡献，可请求维护者分配任务给您。任务分配后，您就可以开始贡献了！

### 开始贡献代码！

任务分配后，您就可以开始编码了！在 Github 或电报上尽可能多地提问。一开始有很多常见问题，但不用担心，我们在这里帮助您。我们真的希望您成为下一个核心贡献者，所以不要害羞，有问题尽管提出 😉。

### 提交 PR

太棒了！看起来您已经解决了问题。接下来，请从您的 fork 提交一个 PR（拉取请求），并请求维护者审核。如果您是开发者，您应该已经熟悉这个阶段了。一旦您的 PR 获得审核通过，且您解决了所有的评论（如果有的话），应该就会被合并！

### 获得收入

恭喜您 🎉。如果您的 PR 已经合并，您有资格通过 [OnlyDust](https://app.onlydust.com/p/madara) 获得支付。请将您的 Github 帐号链接到 OnlyDust，您应该会在一两周内收到您的付款。

</Steps>
