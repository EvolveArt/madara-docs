# Madara EVM

尽管我们坚信 Cairo 不仅是一种高效的证明语言，而且还是一种先进的智能合约语言（详细信息[请点击这里](./cairo)），但为何还要在 EVM 上部署，原因如下：

1. 已经有了经过审计的智能合约，希望能够复用
2. 团队已经具备 Solidity 经验，并希望快速推出产品
3. 随着时间的推移，EVM 也在不断改进，可能还不想放弃使用它

或者也许您有其他完全合理的理由使用 EVM — 使用 Madara，这是完全可行的！正如我们所说的，Madara 是一个高度可定制的技术栈。如果您需要 EVM 兼容性，您可以在 Madara 的基础上利用 [Kakarot](https://www.kakarot.org/) 技术栈。Kakarot 本质上是用 Cairo 编写的 zkEVM。这意味着您的 EVM 代码实际上还是在 Cairo 中执行的，但对于最终用户和智能合约开发者来说，感觉就像在以太坊上一样！
